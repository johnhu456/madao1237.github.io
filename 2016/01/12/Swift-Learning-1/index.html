Swift Learning Note(1)

类型变量

let 定义为常量
var 定义才为变量
当类型不明确时，可以在变量名后+冒号+类型
如下 

var count:Double = 70 或者 var count:Double

Swift 不会做隐式转换，显式转换类型如 

let width = 94
let widthLabel = String(width)

针对数组，var声明的时候即生成一个可变数组，字典同样。
声明指定元素类型的数组或字典，格式如下 

let emptyArray = [String]()
let emptyDictionary = [String: Float]()

“如果我们同时需要每个数据项的值和索引值，可以使用enumerate()方法来进行数组遍历。enumerate()返回一个由每一个数据项索引值和数据值组成的元组。我们可以把这个元组分解成临时常量或者变量来进行遍历：

for (index, value) in shoppingList.enumerate() {
    print("Item \(String(index + 1)): \(value)")
}”

对于可变集合，可以用var someSet = Set() 或者 var someSet：Set = [“1”,”2”,”3”]来创建

控制语句

循环语句差别不大，语法稍微简洁了些，do-while语句改成了repeat—while。值得注意的是 if 后面必须为BOOL类型， 意味着if （int）等的形式将不再有用。

for语句可以用如此表示： 

for i in 0..<4{
    print(i)
}
for i in 0...4{
    print(i)
}

前者不包含上届，后者…表示包含上界。
遍历字典的方式： 

var keys = [String]()
var values = [String]()
var dict1 = ["name": "mike", "age": "18"];
for (name, age) in dict1 {
  keys.append(name)
  values.append(age)
}

字符串中的字符也可以遍历 

for character in "Hello" {
      println(character)
  }
  // H
  // e
  // l
  // l
  // o

Tips

  • Swift中的结构体和类基本相同，结构体也可以有自己的方法，变量。
    但是结构体不能继承，并且在值传递过程中，结构体传递的是值，而类传递的是引用。
  • switch语句中，比如 case webtype:(let type）{},代表type此时相当于switch变量。

  • Swfit中 传递selector参数时，不需要像OC那样@Selector() ,直接传函数名字符串即可： 

    myTimer = NSTimer.scheduledTimerWithTimeInterval(0.1, target: self, selector:"updateTime", userInfo: nil, repeats: true)

  • Swift中数字转换成字符串的几种方法：

    • 直接转换 ： 

      var f = 123
var s:String = "\(f)" //123  或者var s = String(f)

    • 保留小数的转换
      var s = String(format: “%.2f”, f)

    • 16进制
      var s:String = String(format: “%x”, i)

  • func多用外部参数名！: 

    func containsCharacter(#string: String, #characterToFind: Character) -> Bool {
       for character in string {

  • Swfit中的错误处理（Throw）
    昨天在使用AVFoundation构造相机的时候，使用如下函数发现始终报错： 

    input = AVCaptureDeviceInput(device: <#T##AVCaptureDevice!#>)throws

    最后发现是throws关键字的问题，Swift2.0以后貌似对这样的函数有强制要求进行错误处理的样子。

今天只学了一点do catch 语句，算是比较简单易懂的部分。
一个示例 ： 

var error: NSError?
do {
    input = try AVCaptureDeviceInput(device: self.getBackCamera())
}
catch let error1 as NSError{
    error = error1
    input = nil
}
if error != nil {
    // do something
}

以上处理错误比较笼统，没有细分，在Swift2.0中，支持了我们自己自定义一些异常，示例代码如下：

1、先自定义错误类型 ：

enum MyError: ErrorType{
       case NotExist
       case Bad
       case Whaterver
   }

3、异常处理： 

var newStr : String?
     var error : NSError?
     do {
        newStr = try self.doSomeThingDangerous()
     }
     catch MyError.Bad{
         //Bad
         newStr = "bad"
     }
     catch MyError.NotExist{
         //
         newStr = "NotExist"
     }
     catch MyError.Whaterver
     {
         //
         newStr = "WhatEver"
     }
     catch let error1 as NSError{
         //
         error = error1
         newStr = "error"
     }
     if error != nil{
          print(newStr)
     }

 }

2、抛出异常的函数 ：

func doSomeThingDangerous() throws -> String{
    if 1==2 {
        throw MyError.Bad
    }
    else if 2==2 {
        throw MyError.NotExist
    }
    else if 3==4 {
        throw MyError.Whaterver
    }
    else{
        return "1423424"
    }
}
