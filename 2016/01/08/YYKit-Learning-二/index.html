KVO学习

苹果官方提供的一种键值观察机制，对遵守KVC的对象有用。对于注册KVO的对象。当对象的键值发生改变时，系统自动发送通知，而作为Observer的对象可以接收到通知，执行用户指定方法。
Key-Value Observing Programming Guide
下面是一些KVO的基本应用（根据官方文档自己写的一点）

Registering as an Observer

对象注册和Observer的注册通知就不多说了，
先贴代码和界面 

@interface KVOToOneViewController()
{
    ValueModel *_instanceModel;
}
@property (weak, nonatomic) IBOutlet UISlider *sldValue;

@property (weak, nonatomic) IBOutlet UILabel *lblValue;

@property (nonatomic, strong) ValueModel *model;


@end

@implementation KVOToOneViewController

- (void)viewDidLoad {
    [super viewDidLoad];
    /**对控件添加KVO*/
    [self.lblValue addObserver:self forKeyPath:@"text" options:NSKeyValueObservingOptionNew|NSKeyValueObservingOptionOld context:nil];
    /**对模型添加KVO*/
    self.model = [[ValueModel alloc] init];
    [self.model addObserver:self forKeyPath:@"value" options:NSKeyValueObservingOptionNew|NSKeyValueObservingOptionOld context:nil];
    /**对于实例变量*/
    _instanceModel = [[ValueModel alloc] init];
    [_instanceModel addObserver:self forKeyPath:@"value" options:NSKeyValueObservingOptionNew|NSKeyValueObservingOptionOld context:nil];
}

- (void)didReceiveMemoryWarning {
    [super didReceiveMemoryWarning];
    // Dispose of any resources that can be recreated.
}
- (IBAction)sldValueChanged:(id)sender {
    UISlider *slider = sender;

    /**通过KVC赋值方式可以触发KVO通知*/
//    [self.lblValue setValue:[NSString stringWithFormat:@"%f",slider.value] forKey:@"text"];
//    [self.model setValue:[NSNumber numberWithFloat:slider.value] forKey:@"value"];
//    [self.model setValue:slider.value];
    /**以下也行*/
//    _model.value = slider.value;

    /**这里直接赋值也可以*/
    self.lblValue.text= [NSString stringWithFormat:@"%f",slider.value];
    self.model.value = slider.value;
    _instanceModel.value = slider.value;
}
- (void)observeValueForKeyPath:(NSString *)keyPath ofObject:(id)object change:(NSDictionary<NSString *,id> *)change context:(void *)context
{
    if ([keyPath isEqualToString:@"text"]) {
        NSLog(@"控件的Change:%@",change);
    }
    else if ([keyPath isEqualToString:@"value"]) {
        NSLog(@"模型的Change:%@",change);
    }
}


在AddObserver中，主要值得注意的是参数Options
NSKeyValueObservingOptionNew = 0x01,
NSKeyValueObservingOptionOld = 0x02,
NSKeyValueObservingOptionInitial = 0x04,
NSKeyValueObservingOptionPrior = 0x08

以上这些参数都决定了在通知启用回调的时候返回的change字典里面的键值。相应的，返回change字典里的Key值如下:
NSKeyValueChangeKindKey;
NSKeyValueChangeNewKey;
NSKeyValueChangeOldKey;
NSKeyValueChangeIndexesKey;
NSKeyValueChangeNotificationIsPriorKey
如果注册时Options没有选择OptionNew，则返回字典里也不会右NewKey
以上代码的Log如下：

    kind = 1;
    new = "0.4661017060279846";
    old = "0.4661017060279846";
}
2016-01-10 17:06:40.206 OCPlayGround[23711:1468999] 控件的Change:{
    kind = 1;
    new = "0.466102";

}

这些Key是干嘛的解释清楚后，再看看各自代表的意思，这篇文章已经讲的很清楚了：整理了一下关于KVO的姿势
关于对通知的触发，试了三个东西，控件，属性，实例变量都可以，好像是用到setter方法的都可以触发，对于不能触发的情况，还没有尝试出。上一篇文章有提到：如果仅仅是改变了某个property所生成的instance variable的值，自动的change notification是不会发送的。
值得注意的是，如果将注册过KVO的对象设为nil，或者直接指向新的一个对象的时候，发生dealloc，程序会崩溃。因为这样会销毁原有对象，但是Observer还在，没有被一起销毁。
所以需要调用RemoveObserver方法。

KVO Compliance

一般来说，KVO通知的发送，都是自动的，但是也可以手动管理KVO通知的发送。
主要是需要重写对象下的automaticallyNotifiesObserversForKey:方法
自动的不说了，主要看一下手动管理。手动的管理，可以有针对性的对Observers发送通知，可以减少不必要的通知发送，也可以将一系列的改变整合为单个通知。需要手动管理时，应该在automaticallyNotifiesObserversForKey:方法里return NO，剩下不需要手动的则调用父类方法。
一个官方的例子: 

+ (BOOL)automaticallyNotifiesObserversForKey:(NSString *)theKey {

    BOOL automatic = NO;
    if ([theKey isEqualToString:@"openingBalance"]) {
        automatic = NO;
    }
    else {
        automatic = [super automaticallyNotifiesObserversForKey:theKey];
    }
    return automatic;  
}

手动管理通知的例子： 

- (void)setOpeningBalance:(double)theBalance {
    [self willChangeValueForKey:@"openingBalance"];
    _openingBalance = theBalance;
    [self didChangeValueForKey:@"openingBalance"];
}

减少不必要的通知： 

- (void)setOpeningBalance:(double)theBalance {
    if (theBalance != _openingBalance) {
        [self willChangeValueForKey:@"openingBalance"];
        _openingBalance = theBalance;
        [self didChangeValueForKey:@"openingBalance"];
    }
}

多重Key值的时候： 

- (void)setOpeningBalance:(double)theBalance {
    [self willChangeValueForKey:@"openingBalance"];
    [self willChangeValueForKey:@"itemChanged"];
    _openingBalance = theBalance;
    _itemChanged = _itemChanged+1;
    [self didChangeValueForKey:@"itemChanged"];
    [self didChangeValueForKey:@"openingBalance"];
}

当对多关系的时候，不仅需要明确值改变的类型，还要明确是哪个index的值发生了改变。改变类型有 NSKeyValueChangeInsertion, NSKeyValueChangeRemoval, or NSKeyValueChangeReplacement. 一个官方例子： 

    - (void)removeTransactionsAtIndexes:(NSIndexSet *)indexes {
        [self willChange:NSKeyValueChangeRemoval
            valuesAtIndexes:indexes forKey:@"transactions"];

    // Remove the transaction objects at the specified indexes.

    [self didChange:NSKeyValueChangeRemoval
        valuesAtIndexes:indexes forKey:@"transactions"];
}

Registering Dependent Keys

还有一种情况就是当一个属性的改变依赖于其他属性的改变的时候，比如全名的改变依赖于姓或者名的改变。例子： 

- (NSString *)fullName {
    return [NSString stringWithFormat:@"%@ %@",firstName, lastName];
}

此时就需要建立起在类中建立起Key的依赖关系。有几种方法。

对一情况下：

方法一：重写keyPathsForValuesAffectingValueForKey 方法 

+ (NSSet *)keyPathsForValuesAffectingValueForKey:(NSString *)key {

    NSSet *keyPaths = [super keyPathsForValuesAffectingValueForKey:key];

    if ([key isEqualToString:@"fullName"]) {
        NSArray *affectingKeys = @[@"lastName", @"firstName"];
        keyPaths = [keyPaths setByAddingObjectsFromArray:affectingKeys];
    }
    return keyPaths;
}

方法二:像Setter和Getter一样，写一个特殊命名的方法：+ (NSSet *)keyPathsForValuesAffecting 

+ (NSSet *)keyPathsForValuesAffectingFullName {
    return [NSSet setWithObjects:@"lastName", @"firstName", nil];
}

对多情况下

keyPathsForValuesAffectingValueForKey方法不适用于对多情况。举个例子：比如你有一个部门对象，里面有很多员工对象，而员工具有薪水属性，当你想要统计部门总薪水的时候，这个属性是和每一个员工的薪水属性相关联的，此时不能简单的重写keyPathsForValuesAffectingValueForKey方法，返回一个employees.salary。
方法一： 

- (void)observeValueForKeyPath:(NSString *)keyPath ofObject:(id)object change:(NSDictionary *)change context:(void *)context {

    if (context == totalSalaryContext) {
        [self updateTotalSalary];
    }
    else
    // deal with other observations and/or invoke super...
}

- (void)updateTotalSalary {
    [self setTotalSalary:[self valueForKeyPath:@"employees.@sum.salary"]];
}

- (void)setTotalSalary:(NSNumber *)newTotalSalary {

    if (totalSalary != newTotalSalary) {
        [self willChangeValueForKey:@"totalSalary"];
        _totalSalary = newTotalSalary;
        [self didChangeValueForKey:@"totalSalary"];
    }
}

- (NSNumber *)totalSalary {
    return _totalSalary;
}

其中[self setTotalSalary:[self valueForKeyPath:@”employees.@sum.salary”]]用到了集合运算符
方法二：
使用CoreData的时候，关联两个类，注册KVO。

KVO内部实现原理：

当某个类的对象第一次被观察时，系统就会在运行期动态地创建该类的一个派生类，在这个派生类中重写基类中任何被观察属性的 setter 方法。

派生类在被重写的 setter 方法实现真正的通知机制，就如前面手动实现键值观察那样。这么做是基于设置属性会调用 setter 方法，而通过重写就获得了 KVO 需要的通知机制。当然前提是要通过遵循 KVO 的属性设置方式来变更属性值，如果仅是直接修改属性对应的成员变量，是无法实现 KVO 的。

同时派生类还重写了 class 方法以“欺骗”外部调用者它就是起初的那个类。然后系统将这个对象的 isa 指针指向这个新诞生的派生类，因此这个对象就成为该派生类的对象了，因而在该对象上对 setter 的调用就会调用重写的 setter，从而激活键值通知机制。此外，派生类还重写了 dealloc 方法来释放资源。 来自[深入浅出Cocoa]详解键值观察（KVO）及其实现机理

KVO学习告一段落。

- (void)addObserverBlockForKeyPath:

这是YY对系统KVO实现的一次封装，主要还是简化了KVO的书写，支持直接Block传入。
实现方面： 

- (void)addObserverBlockForKeyPath:(NSString *)keyPath block:(void (^)(__weak id obj, id oldVal, id newVal))block {
    if (!keyPath || !block) return;
    _YYNSObjectKVOBlockTarget *target = [[_YYNSObjectKVOBlockTarget alloc] initWithBlock:block];
    NSMutableDictionary *dic = [self _yy_allNSObjectObserverBlocks];
    NSMutableArray *arr = dic[keyPath];
    if (!arr) {
        arr = [NSMutableArray new];
        dic[keyPath] = arr;
    } 
    [arr addObject:target];
    [self addObserver:target forKeyPath:keyPath options:NSKeyValueObservingOptionNew | NSKeyValueObservingOptionOld context:NULL];
}
_YYNSObjectKVOBlockTarget

一个自定义的Target类，当调用AddObserver时会自动创建一个这个类型的Target类，用来对KVO通知做响应，这样就能够实现注册KVO时传入Block。在_YYNSObjectKVOBlockTarget中的实现如下: 

- (void)observeValueForKeyPath:(NSString *)keyPath ofObject:(id)object change:(NSDictionary *)change context:(void *)context {
    if (!self.block) return;

    BOOL isPrior = [[change objectForKey:NSKeyValueChangeNotificationIsPriorKey] boolValue];
    if (isPrior) return;

    NSKeyValueChange changeKind = [[change objectForKey:NSKeyValueChangeKindKey] integerValue];
    if (changeKind != NSKeyValueChangeSetting) return;

    id oldVal = [change objectForKey:NSKeyValueChangeOldKey];
    if (oldVal == [NSNull null]) oldVal = nil;

    id newVal = [change objectForKey:NSKeyValueChangeNewKey];
    if (newVal == [NSNull null]) newVal = nil;

    self.block(object, oldVal, newVal);
}

因为平常情况下KVO都是对单，并且我们需要的选项也不多，所以YY在这里只保留了Setting，NewKey和OldKey选项。
符合条件后，执行Block；

获取观察者： 

- (NSMutableDictionary *)_yy_allNSObjectObserverBlocks {
    NSMutableDictionary *targets = objc_getAssociatedObject(self, &block_key);
    if (!targets) {
        targets = [NSMutableDictionary new];
        objc_setAssociatedObject(self, &block_key, targets, OBJC_ASSOCIATION_RETAIN_NONATOMIC);
    }
    return targets;
}

用到了前面RunTime关联对象的知识，将被关联对象和观察者关联起来，这样可以拥有多个Observr，同时Block又不会冲突。
