用Hexo+Github+Markdown搭建自己的博客(Mac)

##前言
初次想要搭建自己的技术博客，也不过是几天前，理由很简单：想把工作中学到的一些东西做个整理并可以分享在互联网。
当然，在此之前完全没有接触过这些东西，所有的搭建工作也都是靠百度东拼西凑，一点点摸索出来。
这是我的第一篇博文，主要整理下自己搭建的过程和遇到的坑。
如果是完完全全的新手，可以按照本文的顺序来，对于那些只有部分不明白的老手来说可以选择性跳过。

##介绍

###Hexo
个人理解就是一个简单的博客框架，在github上开源，我不懂Node.js，没做过深入研究。
在Github上的地址： https://github.com/hexojs/hexo

###Github
程序员中最受欢迎的代码托管平台，蛮好用的，也提供免费得博客搭建。我现在用的还不多，日后相信经常会用到。

###Markdown
Markdown是一种可以使用普通文本编辑器编写的标记语言，通过简单的标记语法，它可以使普通文本内容具有一定的格式。目前正在使用，方便简单优美。

##教程

###使用Github

  • 注册
    点击Github进入官网进行注册

    找到下图区域

    从上到下依次填写用户名、邮箱地址、密码,再点击绿色按钮注册

    正常的话会看到如下页面，遇到网络问题耐心多试试。Github经常难以访问。

    本文主要介绍如何用Github搭建博客，所以其他功能不做多得介绍了。直接点击Finish Up。
    需要邮箱验证，自行验证一下就好了。

  • 建立Github仓库
    在下一个页面中寻找下图区域，点击绿色按钮(新建一个Github仓库)

    按下图勾选

    注意黑色打码部分，就针对做博客来说，起名强制规定，其余做代码托管没有命名规定。

    Repository name必须和Owner一致，比如Owner为zhangsan，Respository name则必须为zhangsan.github.io，这个以后也将作为访问博客的域名。
    下面Initialize With README 可选可不选。
    点击绿色按钮后仓库创建完成。

  • 生成SSH KEY
    Git是分布式的代码管理工具，远程的代码管理是基于SSH的，所以要使用远程的Git则需要SSH的配置。
    以下代码都在终端中进行操作。

    • 1 检查SSH：

      npm install -g hexo

      sudo npm install -g hexo

      hexo init

      我遇到了如下提示： ***You are almost done! Don't forget to run 'npm install' before you start blogging with Hexo!***  
按照提示输入:```npm install

      生成文件：
      hexo generate
      最后，在本地跑一下hexo自动生成的网页吧：
      hexo server
      成功后如下提示：Hexo is running at http://0.0.0.0:4000/. Press Ctrl+C to stop. (Ctrl + C 可以停止Server)
      浏览器中输入：http://0.0.0.0:4000/ 看到下图就表示成功了
